How the feedback loop works

1

You judge

cte reward / cte punish "reason" — from the terminal or the menu-bar overlay (⌘⇧G / ⌘⇧B).

2

The ledger records

An atomic JSON ledger tracks every token, your balance, grade and streak — stored in ~/.claude-token-economy/.

3

Claude sees it

Claude Code hooks (SessionStart / UserPromptSubmit) inject a compact summary into Claude's context, so it adapts on the very next prompt.

What's in the box

Hooks inject balance + recent reasons into Claude's context. The whole point.

Get started

git clone https://github.com/0xcnr0/claude-token-economy
cd claude-token-economy
npm install

# wire the hooks into Claude Code (backs up your settings first)
node packages/core/bin/cte.js install-hooks

# judge your agent
cte reward "wrote thorough tests"
cte punish "skipped the edge cases"

# launch the menu-bar overlay (⌘⇧G reward · ⌘⇧B punish)
npm run overlay

Then start a new Claude Code session and ask it what its token-economy standing is. It'll know.
